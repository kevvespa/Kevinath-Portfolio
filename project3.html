    • Trip Reservation System

    An online trip reservation system enabling customers to book trips, send requests, and access information anytime without relying on business hours. The system replaces manual processes, allowing managers to handle trip requests, view customer details, allocate staff, and notify coaches and drivers of schedules and announcements, ultimately improving efficiency, customer experience, and business operations for Fun Travel.

    Project Aim and Goal

    The entire project was involved with four (4) main developers throughout the planning, analysis, design, implementation and support phase of the project.The aim of this project was to design and develop a web-based trip reservation system for Fun Travel Company to replace inefficient manual processes and eliminate service limitations caused by restricted operating hours. The system enables customers to register, book trips, make changes, and communicate with staff anytime online. It streamlines internal operations by allowing managers to manage trip requests, allocate staff based on real-time availability, and maintain updated trip records. Drivers and coaches can manage their schedules directly through the platform. Overall, the project enhances customer convenience, improves operational efficiency, reduces human error, and boosts business growth.


    Stack and Explanation

    Stack: HTML, CSS, JavaScript, PHP, Microsoft SQL Server, Apache Web Server, Visual Studio Code, HTTPS Protocol.
    Explanation: The Fun Travel web application was developed using HTML, CSS, JavaScript, and PHP within the Visual Studio Code environment. Apache was utilized as the web server, and Microsoft SQL Server was implemented for managing customer and business data securely. The application is secured using HTTPS to ensure safe data transmission for all users. Designed to be fully responsive, the system operates seamlessly across desktops and smartphones with Internet access. It supports customer trip reservations, staff and manager trip management, and enforces cookie consent for compliance and user tracking. The project was completed under a RM 30,000 budget and within a 3-month timeline, with a development team of four handling all project phases from planning to support.
